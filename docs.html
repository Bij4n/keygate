Quickstart

Get Keygate running locally in under 5 minutes.

Prerequisites

  • Node.js 20+
  • Docker (for PostgreSQL and Redis) — or use the in-memory demo mode

Option 1: Demo Mode (No Dependencies)

The fastest way to try Keygate. Runs entirely in-memory.

git clone https://github.com/Bij4n/keygate.git
cd keygate
npm install
npm run build --workspace=packages/core
npx tsx packages/server/src/demo.ts

Open http://localhost:3100/preview.html for the dashboard.

Option 2: Full Setup

git clone https://github.com/Bij4n/keygate.git
cd keygate
npm install

# Start PostgreSQL and Redis
docker compose up -d postgres redis

# Configure
cp .env.example .env
# Edit .env with your encryption key and JWT secret

# Run migrations
npm run migrate --workspace=packages/server

# Start server + dashboard
npm run dev

Create Your First Connection

# Register
curl -X POST http://localhost:3100/api/auth/register \
  -H 'Content-Type: application/json' \
  -d '{"email":"you@example.com","password":"your-password"}'

# Create a connection
curl -X POST http://localhost:3100/api/connections \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{"provider":"github","scopes":["repo:read"]}'

# Issue a scoped token
curl -X POST http://localhost:3100/api/tokens/issue \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{"connectionId":"conn_...","agentId":"my-agent","scopes":[{"resource":"repos","actions":["read"]}],"ttl":3600}'

Architecture

Keygate is organized as a monorepo with 5 packages.

PackageDescription
@keygate/coreVault logic, encryption, providers, scope validation, anomaly detection, policy engine, agent registry
@keygate/serverExpress REST API with auth, rate limiting, audit logging
@keygate/dashboardNext.js web dashboard
@keygate/mcpMCP server for agent integration
@keygate/sdkTypeScript client SDK

Credential Flow

Keygate uses envelope encryption. Your master credentials are encrypted with a per-connection key, which is itself encrypted with your account's root key.

Root Key (from ENCRYPTION_KEY + user salt)
  └─ Connection Key (per OAuth connection)
       └─ Stored Credential (encrypted at rest)
            └─ Scoped Token (short-lived, limited permissions)

Token Lifecycle

  1. Agent requests a token via SDK or MCP tool, specifying scopes and TTL
  2. Keygate checks policies and agent permissions
  3. If approved, issues a scoped token with an opaque reference
  4. Agent uses the reference to resolve the actual access token (each resolution is logged)
  5. Token auto-expires after TTL, or can be revoked instantly

Security Model

Keygate is designed around the principle that agents are untrusted callers. Security is enforced architecturally, not by prompts.

Defense in Depth

1. Opaque References

Agents never see actual credentials. They receive opaque references (ref_...) that the broker resolves at execution time. If an agent's context window is compromised via prompt injection, the attacker gets a reference — not a usable token.

2. Scoped, Short-Lived Tokens

Every token is scoped to specific resources and actions, with a TTL (default: 1 hour, max: 24 hours). Even if a token is stolen, it expires quickly and can only do what it was scoped for.

3. Anomaly Detection

The anomaly engine analyzes every audit entry in real-time against 6 rules:

RuleWhat It Detects
Velocity SpikeAgent issues too many tokens in a short window
Off-Hours AccessCredential requests outside business hours
Scope EscalationAgent repeatedly requests higher permissions
Unknown AgentUnregistered agent attempts access
Bulk Data AccessExcessive token resolutions (possible exfiltration)
Failed Auth BurstMultiple failed authentication attempts

4. Policy Engine

Define rules that gate access. Policies can deny requests or require human approval based on conditions like agent identity, provider, scope actions, time windows, and TTL limits.

5. Human-in-the-Loop

High-risk operations trigger approval requests. The user must approve before the agent receives access. This stops confused deputy attacks cold — even if an agent is tricked via prompt injection, it can't act without human authorization.

6. Agent Identity & Trust

Every agent gets a registered identity with a trust score (0-100). Trust builds with successful operations and degrades on failures or anomalies. Agents that drop below a trust threshold are auto-suspended.

Encryption

All credentials are encrypted at rest using AES-256-GCM with per-connection derived keys. Token values are stored with both a SHA-256 hash (for lookups) and an encrypted copy (for resolution).

Important: The ENCRYPTION_KEY environment variable is the root of your encryption hierarchy. Rotate it carefully — all stored credentials must be re-encrypted.

TypeScript SDK

The @keygate/sdk package provides a typed client for integrating Keygate into any agent.

Installation

npm install @keygate/sdk

Basic Usage

import { KeygateClient } from '@keygate/sdk';

const kg = new KeygateClient({
  apiKey: process.env.KEYGATE_API_KEY,
  agentId: 'my-agent',
});

// Request a scoped token
const { accessToken, tokenId } = await kg.getToken(
  'conn_github_abc',
  [{ resource: 'repos', actions: ['read'] }],
  { ttl: 3600 }
);

// Use it
const repos = await fetch('https://api.github.com/user/repos', {
  headers: { Authorization: `Bearer ${accessToken}` },
});

// Revoke when done
await kg.revokeToken(tokenId);

withToken Helper

Auto-revokes the token after use:

const repos = await kg.withToken(
  'conn_github_abc',
  [{ resource: 'repos', actions: ['read'] }],
  async (accessToken) => {
    const res = await fetch('https://api.github.com/user/repos', {
      headers: { Authorization: `Bearer ${accessToken}` },
    });
    return res.json();
  }
);

MCP Server

Keygate ships as an MCP server, allowing any MCP-compatible agent to request credentials.

Configuration

Add Keygate to your MCP config:

{
  "mcpServers": {
    "keygate": {
      "command": "npx",
      "args": ["@keygate/mcp"],
      "env": {
        "KEYGATE_API_URL": "http://localhost:3100",
        "KEYGATE_API_KEY": "kg_key_your_api_key"
      }
    }
  }
}

Available Tools

ToolDescription
keygate_list_connectionsList active service connections
keygate_request_tokenRequest a scoped, time-limited token
keygate_resolve_tokenResolve a reference to an actual token
keygate_revoke_tokenRevoke an active token
keygate_list_providersList available service providers
keygate_audit_logView recent audit entries
Note: The MCP server uses the ApiKey auth scheme. Generate an API key from the dashboard Settings page or via POST /api/auth/api-keys.

Self-Hosting

Keygate can be self-hosted with Docker Compose or deployed to any Node.js hosting platform.

Docker Compose

git clone https://github.com/Bij4n/keygate.git
cd keygate
cp .env.example .env

# Generate secure keys
echo "ENCRYPTION_KEY=$(openssl rand -hex 32)" >> .env
echo "JWT_SECRET=$(openssl rand -hex 32)" >> .env

# Start everything
docker compose up -d

Environment Variables

VariableRequiredDescription
DATABASE_URLYesPostgreSQL connection string
REDIS_URLNoRedis connection string (for rate limiting)
ENCRYPTION_KEYYes32-byte hex key for credential encryption
JWT_SECRETYesSecret for signing JWT tokens
PORTNoServer port (default: 3100)
DASHBOARD_URLNoDashboard URL for CORS (default: http://localhost:3200)
Security: Always generate unique ENCRYPTION_KEY and JWT_SECRET values for production. Never commit them to version control.

API: Authentication

Register

POST/api/auth/register
{
  "email": "user@example.com",
  "password": "min-8-chars",
  "name": "Optional Name"
}

Returns a JWT token and user object.

Login

POST/api/auth/login
{
  "email": "user@example.com",
  "password": "your-password"
}

Generate API Key

POST/api/auth/api-keys

Requires Authorization: Bearer TOKEN

{
  "name": "production-agent",
  "scopes": ["*"]
}

Returns the raw API key (shown only once). Use as Authorization: ApiKey kg_key_...

API: Connections

GET/api/connections

List all connections for the authenticated user.

GET/api/connections/:id

Get a specific connection.

POST/api/connections
{
  "provider": "github",
  "scopes": ["repo:read", "read:user"],
  "metadata": {}
}
DELETE/api/connections/:id

Revokes the connection and all associated tokens.

API: Tokens

POST/api/tokens/issue
{
  "connectionId": "conn_...",
  "agentId": "my-agent",
  "scopes": [{"resource": "repos", "actions": ["read"]}],
  "ttl": 3600,
  "maxUsage": 10
}

Returns tokenId, reference, expiresAt.

POST/api/tokens/resolve
{"reference": "ref_..."}

Returns the actual accessToken. Each resolution is logged.

DELETE/api/tokens/:tokenId

Revokes a token immediately.

GET/api/tokens/connection/:connectionId

List active tokens for a connection.

API: Agents

GET/api/agents

List registered agents.

POST/api/agents
{
  "name": "coding-assistant",
  "description": "Helps with code reviews",
  "permissionBoundary": {
    "allowedProviders": ["github"],
    "maxTokenTTL": 3600,
    "maxConcurrentTokens": 5,
    "allowedActions": ["read"],
    "requireApproval": false
  }
}
POST/api/agents/:id/suspend

Suspend an agent (blocks all token requests).

POST/api/agents/:id/activate

Re-activate a suspended agent.

API: Policies

GET/api/policies

List all policies.

POST/api/policies
{
  "name": "No write access after hours",
  "effect": "deny",
  "conditions": [
    {"type": "scope_action", "actions": ["write", "delete"]},
    {"type": "time_window", "startHour": 9, "endHour": 18, "days": [1,2,3,4,5]}
  ]
}

Condition Types

TypeFieldsDescription
agentagentIdsMatch specific agent IDs
providerprovidersMatch specific providers
scope_actionactionsMatch requested actions (read/write/delete/admin)
time_windowstartHour, endHour, daysMatch requests outside the specified window
max_ttlmaxSecondsMatch requests exceeding TTL limit
resourceresourcesMatch specific resources

Effects

  • deny — Block the request immediately
  • require_approval — Hold for human approval
  • allow — Explicitly allow (overrides lower-priority denies)
